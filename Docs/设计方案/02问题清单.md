# 问题清单（以提示词为准）                                                                                                                                             
- 存档日期：2026-03-06                                                                                                                                                 
- 约束基线：以 `backend/llm_agent/prompts` 的提示词语义为最高优先级                                                                                                    
- 当前口径：`mine` 同时覆盖挖矿与砍树，不允许 `chop_tree`                                                                                                              
                                                                                                                                                                        
## P0 问题（阻断主流程）                                                                                                                                               
1. 主运行入口未接入 LLM + LangGraph 主链路                                                                                                                             
现状：`backend/main.py` 仍是 minimal 直连模式，`player_message` 未调用 `graph/workflow`。                                                                              
影响：提示词、Router、Planner、Knowledge Loader 实际不生效。                                                                                                           
                                                                                                                                                                        
2. 执行层能力与规划层动作不闭环                                                                                                                                        
现状：规划层会产出 `mine/craft/place/pick_up...`，但当前可见执行入口未形成完整任务执行管线。                                                                           
影响：会出现“能规划不能执行”。                                                                                                                                         
                                                                                                                                                                        
## P1 问题（高概率运行异常）
1. Task Planner 提示词示例结构与 Pydantic 输出结构不一致                                                                                                               
现状：示例是裸动作列表，不是 `{"plan":[...]}`。                                                                                                                        
影响：结构化输出解析失败概率高，频繁回退到 Router 单任务。                                                                                                             
                                                                                                                                                                        
2. chat 分支回复未进入业务回传链路                                                                                                                                     
现状：`reply_text` 主要是打印，未统一进入对插件回传的数据结构。                                                                                                        
影响：聊天分支体验不稳定，行为不可追踪。                                                                                                                               
                                                                                                                                                                        
3. `env_snapshot` 缺少稳定生产节点                                                                                                                                     
现状：Planner 读取 `env_snapshot`，但主流程缺少固定注入来源。                                                                                                          
影响：规划依赖的环境上下文不完整。                                                                                                                                     
                                                                                                                                                                        
## P2 问题（一致性与可维护性）                                                                                                                                         
1. 文档历史状态冲突                                                                                                                                                    
现状：旧 `backend/README.md` 描述 “no LLM planning”，与当前架构推进方向冲突。                                                                                          
处理：删除旧文档，后续以 `Docs/设计方案` 为主。                                                                                                                        
                                                                                                                                                                        
2. 动作命名跨层需要统一词表                                                                                                                                            
现状：`pick_up` 与 `pickup` 等历史命名并存风险。                                                                                                                       
影响：后续接线时容易出现字段映射错误。                                                                                                                                 
                                                                                                                                                                        
## 已确认变更                                                                                                                                                          
1. 动作口径统一：仅保留 `mine`，不使用 `chop_tree`。                                                                                                                   
2. 知识装载路径口径：`backend/llm_agent/prompts/knowledge/`。                                                                                                          
3. 路由规则口径：`intent=chat` 时 `required_knowledge` 必须是 `[]`。                                                                                                   
                                                                                                                                                                        
## 建议落地顺序                                                                                                                                                        
1. 先打通主入口到 LangGraph 链路（router -> knowledge_loader -> task_planner -> enqueue）。                                                                            
2. 再做动作执行映射闭环（task_queue -> translator/executor -> mineflayer）。                                                                                           
3. 最后补强环境快照注入与 chat 回传链路。
