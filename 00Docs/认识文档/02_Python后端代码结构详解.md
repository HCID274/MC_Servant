# Python 后端代码结构详解

这份文档将为你展示 `MC_Servant/backend` 目录下的详细代码结构。通过这个可视化的树状图，你可以快速定位到你想修改的功能模块。

## 📂 Backend (根目录)

| 文件名 | 作用 |
| :--- | :--- |
| `main.py` | **程序的入口**。启动 FastAPI 服务器，通过 WebSocket 连接 Java 插件。 |
| `protocol.py` | **通信字典**。定义了 Python 和 Java 之间传输的数据格式（如 `MessageType`）。 |
| `config.py` | **配置文件**。读取环境变量（.env），管理数据库地址、Bot 密码等设置。 |

---

## 📂 bot/ (机器人控制层)
> **职责**：负责控制机器人的“身体”，执行具体动作。

| 文件名 | 作用 |
| :--- | :--- |
| `mineflayer_adapter.py` | **核心驱动**。通过 `javascript` 库调用 Node.js 的 Mineflayer，是真正的“方向盘”。 |
| `actions.py` | **动作库**。封装了高级动作，如 `mine_block`（自动寻路挖掘）、`craft`（自动合成）。 |
| `lifecycle_manager.py` | **保姆**。负责 Bot 的上线、下线、重启以及超时踢出机制。 |
| `tag_resolver.py` | **字典**。解决 Minecraft 标签对应问题，比如知道 `minecraft:logs` 包含了橡木、桦木等。 |
| `interfaces.py` | **接口定义**。定义了 Bot 应该具备的基本功能标准。 |

---

## 📂 task/ (任务系统层)
> **职责**：机器人的“前额叶”，负责计划、执行和反思。

| 文件名 | 作用 |
| :--- | :--- |
| `executor.py` | **执行官**。维护任务栈，驱动整个“规划-执行”循环。 |
| `llm_planner.py` | **军师**。调用 LLM 将自然语言（“去挖铁”）拆解为具体的任务参数。 |
| `universal_runner.py` | **全能跑手**。实际执行栈中任务的逻辑，集成了移动、挖掘、放置等多种能力。 |
| `stack_planner.py` | **任务栈**。一个先进后出的数据结构，用于管理主任务和子任务（前置任务）。 |
| `prerequisite_resolver.py` | **故障检修员**。当任务失败（如缺工具）时，它分析原因并生成补救任务（如制作工具）。 |
| `decomposer.py` | **拆解器**。将一个宏大的目标（“盖房子”）拆解为一系列小步骤。 |
| `recovery_coordinator.py` | **救援队**。当 Bot 卡住或发生严重错误时，负责接管控制权进行垂直挖掘脱困。 |
| `dynamic_resolver.py` | **备用大脑**。当预设逻辑无法解决问题时，尝试询问 LLM 该怎么办（慢速路径）。 |
| `behavior_rules.py` | **行为准则**。定义了 Bot 的一些基础行为规则和限制。 |

---

## 📂 state/ (状态管理层)
> **职责**：机器人的“潜意识”，管理当前处于什么模式。

| 文件名 | 作用 |
| :--- | :--- |
| `machine.py` | **状态机**。核心引擎，管理状态切换（如从 Idle 切换到 Working）。 |
| `states.py` | **具体状态**。定义了 `IdleState`（发呆）、`WorkingState`（干活）、`ChattingState`（聊天）等逻辑。 |
| `context.py` | **上下文**。存储运行时的临时记忆，比如当前正在聊天的玩家是谁。 |
| `permission.py` | **权限管理**。决定谁是主人，谁可以指挥 Bot。 |

---

## 📂 llm/ (大模型层)
> **职责**：处理所有与 AI 智能相关的请求。

| 文件名 | 作用 |
| :--- | :--- |
| `factory.py` | **工厂**。根据配置创建不同的大模型客户端（如 OpenAI, Qwen 等）。 |
| `context_manager.py` | **记忆管家**。管理对话历史，负责压缩旧记忆，防止 Token 溢出。 |
| `intent.py` | **意图识别**。判断玩家的一句话是“闲聊”还是“下指令”。 |
| `personality.py` | **人设**。定义 Bot 的性格 prompt（比如是傲娇还是呆萌）。 |

---

## 📂 perception/ (感知层)
> **职责**：处理游戏环境数据，告诉 Bot 周围有什么。

| 文件名 | 作用 |
| :--- | :--- |
| `scanner.py` | **雷达**。扫描周围的实体、方块，寻找掉落物或特定矿物。 |
| `inventory.py` | **背包管理**。统计背包里的物品，判断材料是否充足。 |
| `knowledge_base.py` | **百科全书**。存储合成配方、方块挖掘等级等静态数据。 |
| `resolver.py` | **物品解析**。模糊匹配物品名称（如玩家说“红石”，解析为 `redstone_dust`）。 |

---

## 📂 websocket/ (网络层)
> **职责**：处理与 Java 插件的实时连接。

| 文件名 | 作用 |
| :--- | :--- |
| `handlers.py` | **路由器**。收到消息后，分发给 LLM 或 状态机 处理。 |
| `connection_manager.py` | **连接池**。管理所有活跃的 WebSocket 连接。 |

---

## 📂 db/ (数据库层)
> **职责**：持久化存储。

| 文件名 | 作用 |
| :--- | :--- |
| `models.py` | **数据表**。定义了 `Player`（玩家信息）和 `Bot`（机器人信息）的表结构。 |
| `database.py` | **DB 连接**。负责连接 SQLite 或其他数据库。 |
