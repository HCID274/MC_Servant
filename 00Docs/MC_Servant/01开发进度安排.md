## ✅ 近期完成 (Week 11-12)

### Phase 3: UniversalRunner + LLM Recovery
- [x] **UniversalRunner MVP**: 统一执行逻辑，兼容 mine/goto/give/craft 等动作。
    - 实现了 "Pure vs Composite" 任务逻辑：纯单步任务（如 "过来"）可由 Python 判定完成，复合任务（如 "做木板"）强制依赖 LLM 确认。
- [x] **执行入口切换**: 全面启用 UniversalRunner，解耦旧版 Executor。
- [x] **LLM 驱动恢复**: 实现 `LLMRecoveryPlanner`，支持 ACT/CLARIFY/ABORT 等策略。
- [x] **行为规则 Hints**: `behavior_hints.txt` 替代硬编码规则，动态注入 Prompt。
- [x] **动作集合对齐**: 消除 UNKNOWN_ACTION，明确动作边界。

### Phase 6: 表演动作系统
- [x] 新增 `spin`, `look_at` 等表演动作接口。
- [x] Chat 模块支持 JSON 响应，分离 Text 与 Actions。
- [x] 实现闲聊时的肢体动作反馈。

### Phase 7: 语义感知与导航
- [x] 新增 `find_location` (语义查找) 与 `patrol` (巡逻) 动作。
- [x] 状态注入 `last_find_location`，支持 "先感知后行动" 策略。
- [x] 解决模糊地理指令 (如 "去山上") 的执行问题。

### Phase 8: 智能工具选择与鲁棒性
- [x] Python 侧基于 `harvestTools` 预判工具需求。
- [x] Fail Fast 机制：缺工具立即报错，触发 LLM 规划 (合成/求助)。
- [x] 移除易崩溃的 `equipForBlock` JS 插件。
- [x] **BFS 砍树**: `mine_tree` 实现 BFS 算法，支持分叉树结构（如樱花树）。
- [x] **手动合成兜底**: `craft` 动作支持 `_manual_craft_generic_sync`，当自动合成失败时尝试手动摆盘。

### Phase 9: 动作反馈优化
- [x] 修复 `goto` 假成功问题，强制检查目标到达。
- [x] 消除动作执行幻觉。
