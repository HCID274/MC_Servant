# MC_Servant 基础架构与技术设计

> **项目定位**: Minecraft 服务器智能 NPC 系统
> **设计原则**: 简单的接口，深度的功能；依赖抽象，而非具体
> **架构状态**: Phase 3+ (UniversalRunner + LLM Recovery)
> **最后更新**: 2026-01-05

---

## 1. 系统概述

MC_Servant 是一个基于 **Neuro-Symbolic (神经符号)** 架构的 Minecraft 智能 NPC 系统。
- **Neuro (神经网络)**: 使用 LLM (如 Qwen) 负责语义理解、意图识别、模糊决策与异常恢复。
- **Symbolic (符号逻辑)**: 使用 Python 负责精确的感知、空间计算、状态机流转与物理执行。

### 核心卖点
1. **采集闭环 (Micro-Loop)**: 启用 Tick Loop 机制，LLM 下达指令后，Python 侧维持执行循环，直到目标完成或遇到不可恢复错误。
2. **神经-符号架构**: LLM 负责“想”，Python 负责“做”。LLM 不处理具体坐标，由 Resolver 层落地。
3. **实时交互**: WebSocket 双向通信，支持自然语言对话与指令。

## 3. 核心机制详解

### 3.1 Universal Tick Loop (通用微观闭环)

打破专用 Runner 模式，采用通用的 **UniversalRunner**。

- **Macro Loop (决策层)**:
    - LLM 接收状态与目标，输出结构化 JSON 指令 (Sub-Goal)。
    - 例: `{"action": "mine", "target": "iron_ore", "limit": 5}`
- **Micro Loop (执行层)**:
    - Python 接管指令，持续驱动 Bot 执行原子动作 (寻路 -> 挖掘 -> 循环)。
    - **Robustness (鲁棒性)**: 引入 `ProgressTimer`，支持进度感知超时（只要 Inventory 有变化或动作有进展，就不会因单一动作耗时过长而超时）。
    - **中断机制**: 高优先级事件 (如血量低、用户指令) 可中断当前 Loop。
    - **Fail Fast**: 遇到问题 (如缺工具) 立即上报，而非死循环重试。

### 3.2 神经-符号感知 (Neuro-Symbolic Perception)

解决 LLM 无法处理空间坐标与模糊指令的问题。

- **语义解析**: LLM 输出语义目标 (如 "log", "highest point")。
- **符号落地 (Resolver)**: Python 侧的 EntityResolver / Scanner 将语义转换为物理坐标。
    - **渐进式扫描**: 默认按 [32, 64] 半径序列进行搜索，优先近处目标，平衡性能与覆盖率。
    - 示例: 用户指令 "去山上" -> LLM 输出 `find_location("highest")` -> Python 扫描地形 -> 返回坐标 `(100, 120, 200)`。

### 3.3 恢复策略 (Recovery Strategy)

采用 **Symbolic Check (Fast Path) -> LLM DynamicResolver (Slow Path)** 策略。

1.  **Symbolic Check (Fast Path)**:
    - 对于简单的、确定性的错误 (如 `NO_TOOL`, `INSUFFICIENT_MATERIALS`)，优先尝试符号化规则判断。
    - 优势: 速度快，成本低，逻辑准确 (如合成表)。
2.  **LLM DynamicResolver (Slow Path)**:
    - 对于复杂场景或未知错误，将上下文 (错误码、状态、可用动作、Tag Knowledge) 喂给 LLM。
    - 策略: Decompose (分解), Insert (插入前置), Replan (重规划), Retry (重试), Escalate (询问)。
    - **Prompt 增强**: 注入 "经验" (Knowledge Base) 而非仅 "状态"，解决如 "樱花木板 vs 橡木木板" 的同义替换问题。

## 4. 关键技术决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| **LLM 主力** | **通义千问 Qwen** | 成本低，中文支持好，适合高频调用。 |
| **通信协议** | **WebSocket** | 保证实时性，避免 HTTP 轮询延迟。 |
| **执行框架** | **UniversalRunner** | 统一所有任务执行流，降低维护成本。 |
| **Bot 驱动** | **Mineflayer (JS)** | 生态最成熟的 MC Bot 库，通过 Python-JS 桥接。 |
| **JSON 处理** | **FastJSON2** | Java 侧高性能 JSON 库。 |
