## 🎯 RL 的作用：API Action 排序优化

### 工作流对比

| 模式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **普通模式** (`normal_step`) | 纯 LLM 决策，根据 prompt 直接调用 API | 灵活性高 | Token 消耗大，速度慢 |
| **RL 模式** (`rl_step`) | RL 推荐动作 → LLM 执行 → 反思结果 → 更新 RL | 速度快，可在线学习 | 需要训练 |

### 执行流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                         RL 模式任务执行流程                           │
├─────────────────────────────────────────────────────────────────────┤
│   1. LLM 接收任务描述 + 环境状态                                      │
│         ↓                                                            │
│   2. [RL Model] PPO 推荐最可能成功的 API 动作                         │
│      - 输入: 当前上下文 (instruction + state + history)              │
│      - 输出: 47 个动作的概率分布                                      │
│         ↓                                                            │
│   3. LLM 根据推荐动作 + 任务描述，调用具体 API                         │
│         ↓                                                            │
│   4. Bot 执行动作 → 得到结果 (observation)                            │
│         ↓                                                            │
│   5. LLM 反思执行结果 → 计算奖励 (reward)                             │
│         ↓                                                            │
│   6. RL 模型更新，学习哪些动作更容易成功                               │
│         ↓                                                            │
│   7. 循环 2-6，直到任务完成或达到最大步数                              │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. PPO Actor-Critic 架构 (`minecraft_ppo.py`)

```python
class Actor(nn.Module):
    """策略网络：文本 → 动作概率分布"""
    
    def __init__(self, llm, tokenizer, hidden_dim: int, action_dim: int):
        super().__init__()
        self.llm = llm  # 使用本地小模型 (Qwen2.5-0.5B)
        hidden_state_dim = llm.config.hidden_size
        
        # MLP 头部：LLM 隐藏状态 → 动作概率
        self.fc = nn.Sequential(
            nn.Linear(hidden_state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim),  # → 47 个动作
            nn.Softmax(dim=-1)
        )

    def forward(self, text: str) -> torch.Tensor:
        """前向传播：文本 → LLM 提取特征 → MLP → 47 个动作的概率"""
        inputs = self.tokenizer(text, return_tensors="pt")
        
        with torch.no_grad():
            outputs = self.llm(input_ids=inputs['input_ids'],
                               output_hidden_states=True)
            hidden_states = outputs.hidden_states[-1]
        
        last_hidden = hidden_states[:, -1, :]  # 最后一个 token
        action_probs = self.fc(last_hidden)
        return action_probs


class Critic(nn.Module):
    """价值网络：文本 → 状态价值"""
    # 结构类似 Actor，但输出维度为 1 (状态价值)
```

### 3. PPO 训练逻辑

```python
class PPO:
    def __init__(self, ...):
        # 使用 Qwen2.5-0.5B 作为特征提取器
        model_name = "/run/determined/NAS1/public/Qwen2.5-0.5B-Instruct-GPTQ-Int8"
        tokenizer, actor, critic = setup_models(model_name, hidden_dim, action_dim)
        
        self.actor = actor
        self.critic = critic
        self.replay_buffer = ReplayBuffer(capacity=10000)
    
    def take_action(self, state, temperature=0.8):
        """根据当前状态选择动作"""
        probs = self.actor(state)
        # 温度缩放 + 噪声增加探索
        scaled_probs = (probs / temperature).softmax(dim=-1)
        noisy_probs = (scaled_probs + torch.rand_like(scaled_probs) * 0.1).softmax(dim=-1)
        action_dist = torch.distributions.Categorical(noisy_probs)
        return action_dist.sample().item()
    
    def train_step(self):
        """PPO 训练步骤"""
        # 1. 从 buffer 采样
        batch = random.sample(self.replay_buffer.buffer, self.batch_size)
        
        # 2. 计算 TD 目标和优势函数
        td_target = rewards + gamma * next_values * (1 - dones)
        advantage = compute_advantage(gamma, lmbda, td_delta)
        
        # 3. 计算 PPO 损失 (clip)
        ratio = exp(log_probs - old_log_probs)
        surr1 = ratio * advantage
        surr2 = clamp(ratio, 1-eps, 1+eps) * advantage
        actor_loss = -mean(min(surr1, surr2))
        
        # 4. 更新网络
        actor_loss.backward()
        critic_loss.backward()
```

### 4. Agent 使用 RL (`pipeline/agent.py`)

```python
class BaseAgent:
    def step(self, task: Task):
        if self.RL_mode != "":
            return self.rl_step(task)  # ← 使用 RL 模式
        else:
            return self.normal_step(task)  # ← 普通 LLM 模式
    
    def rl_step(self, task: Task):
        max_rl_steps = 5
        
        while max_rl_steps > 0:
            # 1. 构建当前上下文
            current_context = f"{instruction}\n{current_state}"
            
            # 2. RL 模型选择动作 (0-46)
            rl_action = self.rl_model.take_action(current_context)
            rl_api = self.rl_env._get_available_actions()[rl_action]
            # 例如: rl_action=1 → rl_api="navigateTo"
            
            # 3. 让 LLM+Bot 执行这个推荐的动作
            (act, obs), detail = self.env.iter_step(
                self.name, 
                current_context,
                recommended_actions=[rl_api]  # ← 传入 RL 推荐的动作
            )
            
            # 4. 通过 LLM 反思结果，计算奖励
            reward, task_status = self.rl_one_step_reflect(
                task.description, task.milestones,
                actions=actions, observations=observations,
                act=act, obs=obs
            )
            
            # 5. 更新 RL 模型
            transition_dict = {
                "states": current_context,
                "actions": rl_action,
                "rewards": reward,
                "next_states": next_context,
                "dones": task_status
            }
            self.rl_model.update(transition_dict)
            self.rl_model.train_step()
            
            max_rl_steps -= 1
```

## 🤔 为什么需要 RL？

| 优势 | 说明 |
|------|------|
| **减少 Token 消耗** | RL 模型直接输出动作概率，无需 LLM 每次生成完整推理 |
| **提升成功率** | 通过在线学习，RL 能学到「某状态下哪个 API 更容易成功」 |
| **加速决策** | Qwen2.5-0.5B + MLP 推理速度远快于大 LLM |
| **经验积累** | Replay Buffer 持久化，可跨会话积累经验 |

## 📝 关键要点

1. **RL 不是端到端的**：RL 只负责「推荐下一步用哪个 API」，具体执行仍由 LLM 完成
2. **奖励来自 LLM 反思**：使用 LLM 评估每步动作的好坏，生成 reward
3. **在线学习**：每执行一步就更新模型，边执行边优化
4. **轻量级**：使用 0.5B 小模型做特征提取，不影响主 LLM 的调用
