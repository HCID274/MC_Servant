## 🏗️ VillagerAgent 核心技术栈

### 1. 任务分解系统 (TaskManager)

```
用户输入: "Build a wooden house with 2 rooms"
           ↓ LLM 分解
┌──────────────────────────────────────────────────────┐
│  Subtask DAG (有向无环图)                              │
│                                                       │
│  [收集木头] ──→ [合成木板] ──→ [建造地基]               │
│                    ↓              ↓                  │
│              [合成门] ──→ [建造墙壁] ──→ [安装门窗]     │
│                              ↓                       │
│                        [建造屋顶]                     │
└──────────────────────────────────────────────────────┘
```

**关键代码**: `TaskManager.init_task()`
- 使用 LLM 将自然语言分解为结构化子任务
- 自动构建依赖关系图 (NetworkX)
- 支持动态任务更新 (update) 和合并 (merge) 两种模式

### 2. Agent 执行引擎 (BaseAgent)

```python
# 三种执行模式
class BaseAgent:
    def step(self, task):
        if self.RL_mode:
            return self.rl_step(task)       # RL 推荐动作 + LLM 执行
        elif self.local_mode:
            return self.local_step(task)    # 本地小模型执行
        else:
            return self.normal_step(task)   # 纯 LLM 决策
```

**normal_step 流程**:
1. 构建包含任务描述/环境状态/历史动作的 Prompt
2. 调用 LLM (GPT-4/Gemini) 生成动作序列
3. 通过 Langchain Agent 执行 Mineflayer API
4. 收集执行结果，调用 `reflect()` 评估任务状态

### 3. 强化学习模块 (PPO)

```
┌─────────────────────────────────────────────────────────┐
│  PPO Actor-Critic 架构                                   │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  输入: "instruction + state + history" (文本)            │
│           ↓                                              │
│  [Qwen2.5-0.5B] → 提取 hidden_states (冻结)              │
│           ↓                                              │
│  [MLP Head] → 47 维动作概率分布                          │
│           ↓                                              │
│  sample() → 推荐动作 (如 "navigateTo")                   │
│           ↓                                              │
│  [GPT-4/Gemini] → 生成具体参数并执行                     │
│           ↓                                              │
│  [LLM Reflect] → 评估结果 → 计算 Reward                  │
│           ↓                                              │
│  [Replay Buffer] → PPO train_step() 更新网络             │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**关键洞察**:
- RL 模型**不直接控制 Bot**，只负责**推荐下一个 API 动作**
- 奖励信号来自 **LLM 的反思评估**，不是游戏内奖励
- 使用小模型 (0.5B) 做特征提取，训练成本低
- 在线学习：每执行 1 步就更新模型

## 🎯 MC_Servant 分层控制架构 (Layered Control Architecture)

> **设计原则**: 简单的接口，深度的功能；依赖抽象，而非具体

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: LLM Planner (大脑)                                 │
│  ├── 输入: 自然语言任务 "帮我挖点铁矿"                        │
│  ├── 输出: JSON 动作序列 [{"action": "mine", "target": ...}] │
│  └── 依赖: ITaskPlanner 抽象接口                             │
└─────────────────────────────┬───────────────────────────────┘
                              ↓ ActionPlan (高层指令)
┌─────────────────────────────────────────────────────────────┐
│  Layer 2: Python Actions (四肢技能)                          │
│  ├── 参数解析 + 语义目标转坐标                               │
│  ├── 错误处理 + 超时控制 + 重试逻辑                          │
│  ├── 返回 ActionResult 给上层反思                            │
│  └── 依赖: IBotActions 抽象接口                              │
└─────────────────────────────┬───────────────────────────────┘
                              ↓ javascript 模块调用
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: Mineflayer Plugins (神经末梢)                      │
│  ├── pathfinder - 寻路导航 + 避障                            │
│  ├── collectblock - 自动采集方块                             │
│  ├── tool - 自动选择正确工具                                 │
│  └── 依赖: Mineflayer Bot 实例                               │
└─────────────────────────────────────────────────────────────┘
```

## 🔧 Phase 1: 底层 - Mineflayer 插件集成 (The Limbs)

### 必须安装的 npm 包

```bash
cd MC_Servant/backend
npm install mineflayer-pathfinder mineflayer-collectblock mineflayer-tool
```

| 插件 | 功能 | 关键能力 |
|------|------|----------|
| `mineflayer-pathfinder` | 寻路导航 | A* 算法、避障、跳跃/游泳 |
| `mineflayer-collectblock` | 方块采集 | 自动寻找 + 路径 + 挖掘 |
| `mineflayer-tool` | 工具选择 | 自动选择最佳工具 |

### 插件加载代码

```javascript
// 在 Bot 初始化时加载插件
const pathfinder = require('mineflayer-pathfinder').pathfinder;
const collectBlock = require('mineflayer-collectblock').plugin;
const toolPlugin = require('mineflayer-tool').plugin;

bot.loadPlugin(pathfinder);
bot.loadPlugin(collectBlock);
bot.loadPlugin(toolPlugin);
```

> [!IMPORTANT]
> **NPM 依赖路径**  
> 务必确保 `npm install` 是在 `MC_Servant/backend/` 目录下执行的，因为 Python 的 `javascript` 库会在**当前运行目录**查找 `node_modules`。

> [!WARNING]
> **Javascript Promise 与 Python Await**  
> `javascript` 库 (JSPyBridge) 处理 JS Promise 有时比较玄学。如果 `await self.bot.collectBlock.collect(target)` 卡住，可能需要用 `asyncio.to_thread` 包装，或者检查 JS 端是否真的返回了 Promise。不过 `mineflayer-collectblock` 通常表现良好。

> [!TIP]
> **Bot 状态的 Token 优化**  
> 在 `get_state()` 中，`inventory` 如果包含大量物品，会占用很多 Token。  
> **优化建议**：将 inventory 简化为合并同类项格式：
> ```json
> {"iron_ore": 64, "cobblestone": 128, "oak_log": 32}
> ```
> 而不是列出每个格子的详细数据。

## 🦾 Phase 2: 中层 - 原子动作封装 (The Skills)

### 核心接口定义 (依赖抽象)

```python
# backend/bot/interfaces.py

from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional, Any, List
from enum import Enum


class ActionStatus(Enum):
    """动作执行状态"""
    SUCCESS = "success"
    FAILED = "failed"
    TIMEOUT = "timeout"
    CANCELLED = "cancelled"


@dataclass
class ActionResult:
    """
    动作执行结果 - 统一的反馈结构
    
    所有动作都必须返回此结构，供 LLM 反思决策
    """
    success: bool
    action: str                    # 执行的动作名
    message: str                   # 人类可读的描述
    status: ActionStatus           # 状态枚举
    data: Optional[Any] = None     # 返回数据（如采集的物品列表）
    error_code: Optional[str] = None  # 错误码
    duration_ms: int = 0           # 执行耗时
    
    # 常见错误码定义
    # INVENTORY_FULL - 背包满了
    # TOOL_BROKEN - 工具损坏
    # TARGET_NOT_FOUND - 找不到目标
    # PATH_BLOCKED - 路径被阻挡
    # TIMEOUT - 超时
    # NO_TOOL - 没有合适工具


class IBotActions(ABC):
    """
    Bot 动作抽象接口
    
    设计原则：
    - 简单的接口：方法参数使用语义化名称，不暴露坐标细节
    - 深度的功能：内部封装寻路、工具选择、错误处理
    - 依赖抽象：上层只依赖此接口，不依赖 Mineflayer 具体实现
    """
    
    @abstractmethod
    async def goto(self, target: str) -> ActionResult:
        """
        导航到目标位置
        
        Args:
            target: 目标位置 (支持格式):
                - 坐标字符串: "100,64,200"
                - 玩家名: "@PlayerName"
                - 语义位置: "spawn", "home", "forest"
        """
        pass
    
    @abstractmethod
    async def mine(self, block_type: str, count: int = 1) -> ActionResult:
        """
        采集指定类型的方块
        
        自动处理：寻找目标、导航、选择工具、挖掘
        
        Args:
            block_type: 方块类型 (如 "iron_ore", "oak_log")
            count: 采集数量
        """
        pass
    
    @abstractmethod
    async def place(self, block_type: str, x: int, y: int, z: int) -> ActionResult:
        """
        在指定位置放置方块
        
        自动处理：导航到附近、选择物品、放置
        """
        pass
    
    @abstractmethod
    async def craft(self, item_name: str, count: int = 1) -> ActionResult:
        """
        合成物品
        
        自动处理：查找配方、检查材料、寻找工作台、合成
        """
        pass
    
    @abstractmethod
    async def give(self, player_name: str, item_name: str, count: int = 1) -> ActionResult:
        """
        将物品交给玩家
        
        自动处理：导航到玩家、丢弃物品
        """
        pass
    
    @abstractmethod
    async def equip(self, item_name: str) -> ActionResult:
        """装备物品到手上"""
        pass
    
    @abstractmethod
    async def attack(self, target: str) -> ActionResult:
        """攻击目标实体"""
        pass
    
    @abstractmethod
    async def scan(self, target_type: str, radius: int = 32) -> ActionResult:
        """
        扫描周围实体/方块
        
        返回: ActionResult.data = [{"name": ..., "position": ...}, ...]
        """
        pass
    
    @abstractmethod
    def get_state(self) -> dict:
        """
        获取 Bot 当前状态
        
        返回: {
            "position": {"x": 0, "y": 64, "z": 0},
            "health": 20,
            "food": 20,
            "inventory": [...],
            "equipped": "diamond_pickaxe"
        }
        """
        pass
```

### 具体实现类

```python
# backend/bot/actions.py

from javascript import require, On
from bot.interfaces import IBotActions, ActionResult, ActionStatus
import asyncio
import logging

logger = logging.getLogger(__name__)


class MineflayerActions(IBotActions):
    """
    Mineflayer 动作实现
    
    封装 javascript 模块调用，提供统一的 Python 异步接口
    """
    
    def __init__(self, bot):
        self._bot = bot  # MineflayerBot 实例
        self._pathfinder = None
        self._mcData = None
        self._setup_plugins()
    
    def _setup_plugins(self):
        """加载 Mineflayer 插件"""
        pathfinder_module = require('mineflayer-pathfinder')
        self._pathfinder = pathfinder_module.pathfinder
        self._goals = pathfinder_module.goals
        self._Movements = pathfinder_module.Movements
        
        collectblock = require('mineflayer-collectblock')
        tool_plugin = require('mineflayer-tool')
        
        self._bot._bot.loadPlugin(self._pathfinder)
        self._bot._bot.loadPlugin(collectblock.plugin)
        self._bot._bot.loadPlugin(tool_plugin.plugin)
        
        # 加载 minecraft-data
        self._mcData = require('minecraft-data')(self._bot._bot.version)
        
        # 配置寻路
        movements = self._Movements(self._bot._bot, self._mcData)
        self._bot._bot.pathfinder.setMovements(movements)
    
    async def goto(self, target: str) -> ActionResult:
        """导航到目标位置"""
        try:
            # 解析目标
            goal = self._parse_target(target)
            if goal is None:
                return ActionResult(
                    success=False,
                    action="goto",
                    message=f"无法解析目标位置: {target}",
                    status=ActionStatus.FAILED,
                    error_code="TARGET_NOT_FOUND"
                )
            
            # 设置目标并等待到达
            self._bot._bot.pathfinder.setGoal(goal)
            
            # 异步等待到达
            await self._wait_for_goal(timeout=30)
            
            return ActionResult(
                success=True,
                action="goto",
                message=f"已到达 {target}",
                status=ActionStatus.SUCCESS,
                data={"target": target}
            )
            
        except asyncio.TimeoutError:
            return ActionResult(
                success=False,
                action="goto",
                message=f"导航到 {target} 超时",
                status=ActionStatus.TIMEOUT,
                error_code="TIMEOUT"
            )
        except Exception as e:
            logger.error(f"goto failed: {e}")
            return ActionResult(
                success=False,
                action="goto",
                message=str(e),
                status=ActionStatus.FAILED,
                error_code="PATH_BLOCKED"
            )
    
    def _parse_target(self, target: str):
        """解析目标字符串为 Goal 对象"""
        # 玩家目标: @PlayerName
        if target.startswith("@"):
            player_name = target[1:]
            player = self._bot._bot.players.get(player_name)
            if player and player.entity:
                return self._goals.GoalFollow(player.entity, 2)
            return None
        
        # 坐标目标: x,y,z
        if "," in target:
            parts = target.split(",")
            if len(parts) == 3:
                x, y, z = map(int, parts)
                return self._goals.GoalBlock(x, y, z)
        
        # 语义位置 (TODO: 扩展)
        return None
    
    async def mine(self, block_type: str, count: int = 1) -> ActionResult:
        """采集方块"""
        try:
            block_id = self._mcData.blocksByName[block_type]
            if not block_id:
                return ActionResult(
                    success=False,
                    action="mine",
                    message=f"未知方块类型: {block_type}",
                    status=ActionStatus.FAILED,
                    error_code="TARGET_NOT_FOUND"
                )
            
            # 使用 collectblock 插件
            collected = await self._collect_blocks(block_id.id, count)
            
            return ActionResult(
                success=True,
                action="mine",
                message=f"成功采集 {len(collected)} 个 {block_type}",
                status=ActionStatus.SUCCESS,
                data={"items": collected, "count": len(collected)}
            )
            
        except Exception as e:
            logger.error(f"mine failed: {e}")
            # 判断具体错误类型
            error_msg = str(e).lower()
            if "inventory" in error_msg:
                error_code = "INVENTORY_FULL"
            elif "tool" in error_msg:
                error_code = "NO_TOOL"
            else:
                error_code = "FAILED"
            
            return ActionResult(
                success=False,
                action="mine",
                message=str(e),
                status=ActionStatus.FAILED,
                error_code=error_code
            )
    
    # ... 其他方法实现 ...
    
    def get_state(self) -> dict:
        """获取 Bot 状态"""
        bot = self._bot._bot
        return {
            "position": {
                "x": int(bot.entity.position.x),
                "y": int(bot.entity.position.y),
                "z": int(bot.entity.position.z)
            },
            "health": bot.health,
            "food": bot.food,
            "inventory": self._get_inventory_summary(),
            "equipped": self._get_equipped_item()
        }
```

## 🧠 Phase 3: 顶层 - 线性任务规划 (The Planner)

### 任务规划器接口

```python
# backend/task/interfaces.py

from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import List, Optional
from bot.interfaces import ActionResult


@dataclass
class ActionStep:
    """单个动作步骤"""
    action: str               # 动作名称
    params: dict              # 动作参数
    description: str = ""     # 人类可读描述


@dataclass 
class ActionPlan:
    """任务执行计划"""
    task_description: str     # 原始任务描述
    steps: List[ActionStep]   # 动作序列
    estimated_time: int = 0   # 预估时间(秒)


@dataclass
class TaskResult:
    """任务执行结果"""
    success: bool
    task_description: str
    completed_steps: List[ActionResult]
    failed_step: Optional[ActionResult] = None
    message: str = ""


class ITaskPlanner(ABC):
    """
    任务规划器抽象接口
    
    职责：将自然语言任务分解为可执行的动作序列
    """
    
    @abstractmethod
    async def plan(self, task_description: str, bot_state: dict) -> ActionPlan:
        """
        规划任务
        
        Args:
            task_description: 自然语言任务描述
            bot_state: Bot 当前状态
            
        Returns:
            ActionPlan: 动作执行计划
        """
        pass
    
    @abstractmethod
    async def replan(
        self, 
        task_description: str, 
        bot_state: dict,
        failed_result: ActionResult,
        completed_steps: List[ActionResult]
    ) -> ActionPlan:
        """
        任务重规划 (执行失败后)
        
        根据失败原因和已完成步骤，生成新的执行计划
        """
        pass
```

### LLM 任务规划器实现

```python
# backend/task/llm_planner.py

from task.interfaces import ITaskPlanner, ActionPlan, ActionStep
from llm.interfaces import ILLMClient
from bot.interfaces import ActionResult
import json
import logging

logger = logging.getLogger(__name__)

# 任务规划 Prompt
PLANNER_SYSTEM_PROMPT = """你是一个 Minecraft 任务规划专家。

给定用户的任务描述和 Bot 当前状态，输出一个 JSON 格式的动作序列。

## 可用动作

| 动作 | 参数 | 说明 |
|------|------|------|
| goto | target: str | 导航到目标 (坐标 "x,y,z" 或玩家 "@name") |
| mine | block_type: str, count: int | 采集方块 |
| place | block_type: str, x: int, y: int, z: int | 放置方块 |
| craft | item_name: str, count: int | 合成物品 |
| give | player_name: str, item_name: str, count: int | 交给玩家 |
| equip | item_name: str | 装备物品 |
| scan | target_type: str, radius: int | 扫描周围 |

## 输出格式

```json
{
  "steps": [
    {"action": "动作名", "params": {...}, "description": "说明"}
  ]
}
```

## 示例

用户: "帮我挖 10 个铁矿石"
输出:
```json
{
  "steps": [
    {"action": "equip", "params": {"item_name": "iron_pickaxe"}, "description": "装备铁镐"},
    {"action": "scan", "params": {"target_type": "iron_ore", "radius": 64}, "description": "搜索铁矿"},
    {"action": "mine", "params": {"block_type": "iron_ore", "count": 10}, "description": "采集10个铁矿"}
  ]
}
```
"""


class LLMTaskPlanner(ITaskPlanner):
    """基于 LLM 的任务规划器"""
    
    def __init__(self, llm_client: ILLMClient):
        self._llm = llm_client
    
    async def plan(self, task_description: str, bot_state: dict) -> ActionPlan:
        """使用 LLM 规划任务"""
        
        user_prompt = f"""## 任务
{task_description}

## Bot 当前状态
位置: {bot_state.get('position', 'unknown')}
生命: {bot_state.get('health', 20)}
背包: {bot_state.get('inventory', [])}
手持: {bot_state.get('equipped', 'empty')}

请输出 JSON 格式的动作序列。
"""
        
        response = await self._llm.chat(
            system_prompt=PLANNER_SYSTEM_PROMPT,
            user_prompt=user_prompt,
            json_mode=True
        )
        
        # 解析 JSON
        plan_data = json.loads(response)
        steps = [
            ActionStep(
                action=step["action"],
                params=step.get("params", {}),
                description=step.get("description", "")
            )
            for step in plan_data.get("steps", [])
        ]
        
        return ActionPlan(
            task_description=task_description,
            steps=steps
        )
    
    async def replan(
        self, 
        task_description: str,
        bot_state: dict,
        failed_result: ActionResult,
        completed_steps: list
    ) -> ActionPlan:
        """失败后重新规划"""
        
        # 构建上下文
        completed_summary = "\n".join([
            f"✅ {r.action}: {r.message}" for r in completed_steps
        ])
        
        user_prompt = f"""## 原始任务
{task_description}

## 已完成步骤
{completed_summary}

## 失败步骤
❌ {failed_result.action}: {failed_result.message}
错误码: {failed_result.error_code}

## Bot 当前状态
{bot_state}

请分析失败原因，输出调整后的动作序列。
"""
        
        response = await self._llm.chat(
            system_prompt=PLANNER_SYSTEM_PROMPT,
            user_prompt=user_prompt,
            json_mode=True
        )
        
        plan_data = json.loads(response)
        steps = [
            ActionStep(
                action=step["action"],
                params=step.get("params", {}),
                description=step.get("description", "")
            )
            for step in plan_data.get("steps", [])
        ]
        
        return ActionPlan(
            task_description=task_description,
            steps=steps
        )
```

### 任务执行器

```python
# backend/task/executor.py

from task.interfaces import ITaskPlanner, TaskResult, ActionPlan
from bot.interfaces import IBotActions, ActionResult
import logging

logger = logging.getLogger(__name__)


class TaskExecutor:
    """
    任务执行器
    
    协调 Planner 和 Actions，驱动任务执行循环
    """
    
    def __init__(
        self, 
        planner: ITaskPlanner,
        actions: IBotActions,
        max_retries: int = 3
    ):
        self._planner = planner
        self._actions = actions
        self._max_retries = max_retries
    
    async def execute(self, task_description: str) -> TaskResult:
        """
        执行任务直到完成或失败
        
        流程:
        1. LLM 规划动作序列
        2. 逐步执行动作
        3. 失败时反思并重规划
        4. 超过重试次数则放弃
        """
        completed_steps = []
        retries = 0
        
        # 1. 初始规划
        bot_state = self._actions.get_state()
        plan = await self._planner.plan(task_description, bot_state)
        
        logger.info(f"Task planned: {len(plan.steps)} steps")
        
        while retries < self._max_retries:
            # 2. 逐步执行
            for step in plan.steps:
                logger.info(f"Executing: {step.action} - {step.description}")
                
                # 执行动作
                result = await self._execute_step(step)
                
                if result.success:
                    completed_steps.append(result)
                    logger.info(f"✅ {step.action}: {result.message}")
                else:
                    # 3. 失败处理
                    logger.warning(f"❌ {step.action}: {result.message}")
                    
                    retries += 1
                    if retries >= self._max_retries:
                        return TaskResult(
                            success=False,
                            task_description=task_description,
                            completed_steps=completed_steps,
                            failed_step=result,
                            message=f"任务失败，已重试 {retries} 次"
                        )
                    
                    # 4. 重规划
                    bot_state = self._actions.get_state()
                    plan = await self._planner.replan(
                        task_description,
                        bot_state,
                        result,
                        completed_steps
                    )
                    logger.info(f"Replanned: {len(plan.steps)} new steps")
                    break  # 重新开始执行新计划
            else:
                # 所有步骤执行完成
                return TaskResult(
                    success=True,
                    task_description=task_description,
                    completed_steps=completed_steps,
                    message="任务完成"
                )
        
        return TaskResult(
            success=False,
            task_description=task_description,
            completed_steps=completed_steps,
            message="任务执行超时"
        )
    
    async def _execute_step(self, step) -> ActionResult:
        """执行单个步骤"""
        action_name = step.action
        params = step.params
        
        # 动态调用对应方法
        action_method = getattr(self._actions, action_name, None)
        if action_method is None:
            return ActionResult(
                success=False,
                action=action_name,
                message=f"未知动作: {action_name}",
                status="failed",
                error_code="UNKNOWN_ACTION"
            )
        
        return await action_method(**params)
```

### 层级职责边界

| 层级 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **Cloud (Big LLM)** | 决策：决定做什么、数量、顺序 | 用户任务 + Bot 状态 | 自然语言规划 |
| **Edge (Small LLM)** | 翻译：将自然语言转为 JSON | 自然语言规划 | JSON 动作序列 |
| **Executor (Python)** | 落地：语义→坐标、调用 API | JSON 动作序列 | ActionResult |

### 设计决策

#### 1. 自然语言格式：半结构化 (Chain of Thought)

```
✅ 推荐格式 (半结构化)：
「计划如下：
1. 首先，我们需要去森林区域。
2. 到达后，采集 20 个橡木原木。
3. 然后，将原木合成为木板。」

❌ 避免格式 (完全自由)：
「去森林砍点木头，大概20块够了。然后做成木板。」

❌ 避免格式 (过度模板化)：
「[移动] 森林 [采集] 橡木原木 × 20」
```

**理由**: 序号帮助小模型对齐动作数量，"首先/然后"帮助理解依赖关系。

#### 2. 语义解析边界

| 语义 | 解析层 | 示例 |
|------|-------|------|
| **数量/物品** | Big LLM | "采集 **20 个橡木原木**" (必须明确) |
| **动作映射** | Small LLM | "采集" → `{"action": "mine"}` |
| **位置解析** | Python | "森林" → 查表或 `bot.findBlock()` |

**理由**: Small LLM (1.5B) 不具备世界知识，无法知道"森林在哪"；也不擅长数学推理，无法计算"盖房子需要多少木头"。

#### 3. 训练数据：逆向合成 (Reverse Synthetic Generation)

```
┌─────────────────────────────────────────────────────────────────────┐
│  Step 1: 定义原子动作 JSON Schema                                    │
│  - goto, mine, craft, place, give, equip, scan, attack              │
└─────────────────────────────┬───────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Step 2: 规则生成合理的 JSON 序列                                    │
│  例如: [goto(forest), mine(oak_log, 20), craft(oak_planks, 80)]     │
└─────────────────────────────┬───────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Step 3: GPT-4/Qwen-Max 反向描述                                     │
│  Prompt: "将以下指令翻译成自然语言，像教别人玩游戏一样分步骤描述"       │
│  Input:  [{"action": "mine", "block": "stone", "count": 10}]        │
│  Output: "1. 首先，找到并挖掘 10 个石头。"                            │
└─────────────────────────────┬───────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Step 4: 构建训练集                                                  │
│  - X: 自然语言描述 (Step 3 输出)                                     │
│  - Y: 原始 JSON 序列 (Step 2 生成，语法保证正确)                     │
│  - 规模: 5000+ 条                                                    │
└─────────────────────────────────────────────────────────────────────┘
```

**优势**: JSON 语法绝对正确（因为是源头），大模型只负责"解释"而非"编程"。

# backend/compiler/interfaces.py

from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import List


@dataclass
class NaturalLanguagePlan:
    """自然语言规划 (NL-IR)"""
    raw_text: str           # 原始文本
    steps: List[str]        # 分步骤列表
    task_description: str   # 原始任务描述


class ICloudPlanner(ABC):
    """云端规划器接口 (Big LLM)"""
    
    @abstractmethod
    async def plan(self, task: str, bot_state: dict) -> NaturalLanguagePlan:
        """生成自然语言规划"""
        pass
    
    @abstractmethod
    async def replan(
        self, 
        task: str, 
        bot_state: dict,
        error: str,
        completed_steps: List[str]
    ) -> NaturalLanguagePlan:
        """失败后重新规划"""
        pass


class IEdgeTranslator(ABC):
    """端侧翻译器接口 (Small LLM)"""
    
    @abstractmethod
    async def translate(self, plan: NaturalLanguagePlan) -> List[dict]:
        """将自然语言规划翻译为 JSON 动作序列"""
        pass
```

### 实现代码

```python
# backend/compiler/cloud_planner.py

from compiler.interfaces import ICloudPlanner, NaturalLanguagePlan
from llm.interfaces import ILLMClient
import re


CLOUD_PLANNER_PROMPT = """你是一个 Minecraft 任务规划专家。

给定用户任务和 Bot 当前状态，输出一个分步骤的执行计划。

## 输出格式要求
1. 以"计划如下："开头
2. 使用数字序号 (1. 2. 3. ...)
3. 每步必须明确：动作、目标物品/位置、数量
4. 使用"首先"、"然后"、"接着"、"最后"等连接词

## 示例

用户: "帮我挖点铁矿"
输出:
计划如下：
1. 首先，装备一把铁镐或石镐。
2. 然后，前往矿洞区域。
3. 接着，采集 10 个铁矿石。
4. 最后，返回到玩家身边，将铁矿石交给玩家。
"""


class CloudPlanner(ICloudPlanner):
    """云端规划器实现"""
    
    def __init__(self, llm: ILLMClient):
        self._llm = llm
    
    async def plan(self, task: str, bot_state: dict) -> NaturalLanguagePlan:
        prompt = f"""## 任务
{task}

## Bot 当前状态
位置: {bot_state.get('position', 'unknown')}
背包: {bot_state.get('inventory', [])}
手持: {bot_state.get('equipped', 'empty')}

请输出分步骤的执行计划。
"""
        response = await self._llm.chat(
            system_prompt=CLOUD_PLANNER_PROMPT,
            user_prompt=prompt
        )
        
        # 解析步骤
        steps = self._parse_steps(response)
        
        return NaturalLanguagePlan(
            raw_text=response,
            steps=steps,
            task_description=task
        )
    
    def _parse_steps(self, text: str) -> List[str]:
        """解析带序号的步骤"""
        pattern = r'\d+\.\s*(.+?)(?=\d+\.|$)'
        matches = re.findall(pattern, text, re.DOTALL)
        return [m.strip() for m in matches if m.strip()]
```

```python
# backend/compiler/edge_translator.py

from compiler.interfaces import IEdgeTranslator, NaturalLanguagePlan
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch
import json


class EdgeTranslator(IEdgeTranslator):
    """端侧翻译器 (本地微调模型)"""
    
    def __init__(self, model_path: str = "models/mc-translator-1.5b"):
        self._tokenizer = AutoTokenizer.from_pretrained(model_path)
        self._model = AutoModelForCausalLM.from_pretrained(
            model_path,
            torch_dtype=torch.float16,
            device_map="auto"
        )
    
    async def translate(self, plan: NaturalLanguagePlan) -> List[dict]:
        prompt = f"""[Instruction]: 将以下计划转换为 JSON 动作序列。

[Plan]:
{plan.raw_text}

[Output]:
"""
        inputs = self._tokenizer(prompt, return_tensors="pt").to(self._model.device)
        
        with torch.no_grad():
            outputs = self._model.generate(
                **inputs,
                max_new_tokens=512,
                temperature=0.1,
                do_sample=False
            )
        
        response = self._tokenizer.decode(outputs[0], skip_special_tokens=True)
        
        # 提取 JSON
        json_str = self._extract_json(response)
        return json.loads(json_str)
    
    def _extract_json(self, text: str) -> str:
        """从响应中提取 JSON 数组"""
        start = text.find('[')
        end = text.rfind(']') + 1
        return text[start:end]
```

# scripts/generate_training_data.py

import json
import random
from typing import List, Dict
import openai  # 或使用 dashscope for Qwen

# 原子动作定义
ACTION_TEMPLATES = {
    "goto": {"target": ["forest", "mine", "village", "home", "@{player}"]},
    "mine": {"block": ["oak_log", "stone", "iron_ore", "diamond_ore"], "count": [5, 10, 20, 64]},
    "craft": {"item": ["oak_planks", "stick", "crafting_table", "iron_pickaxe"], "count": [1, 4, 16, 64]},
    "place": {"block": ["oak_planks", "stone", "torch"], "x": "random", "y": "random", "z": "random"},
    "give": {"target": ["{player}"], "item": ["oak_log", "iron_ore", "diamond"], "count": [1, 10, 64]},
    "equip": {"item": ["iron_pickaxe", "diamond_sword", "shield"]},
    "scan": {"target": ["iron_ore", "diamond_ore", "chest", "villager"], "radius": [16, 32, 64]},
}


def generate_random_sequence(length: int = 3) -> List[Dict]:
    """生成随机合理的动作序列"""
    actions = []
    for _ in range(length):
        action_name = random.choice(list(ACTION_TEMPLATES.keys()))
        template = ACTION_TEMPLATES[action_name]
        params = {}
        for key, values in template.items():
            if values == "random":
                params[key] = random.randint(-100, 100)
            elif isinstance(values, list):
                params[key] = random.choice(values)
        actions.append({"action": action_name, **params})
    return actions


def reverse_describe(actions: List[Dict], client) -> str:
    """使用大模型反向描述 JSON 为自然语言"""
    prompt = f"""将以下 Minecraft 机器指令转换为自然语言计划。

要求：
1. 以"计划如下："开头
2. 使用数字序号
3. 使用"首先"、"然后"等连接词
4. 像教别人玩游戏一样描述

指令:
{json.dumps(actions, ensure_ascii=False, indent=2)}

自然语言计划:
"""
    response = client.chat.completions.create(
        model="qwen-max",
        messages=[{"role": "user", "content": prompt}]
    )
    return response.choices[0].message.content


def generate_dataset(num_samples: int = 5000):
    """生成训练数据集"""
    client = openai.OpenAI(...)  # 配置 API
    
    dataset = []
    for i in range(num_samples):
        # 生成 JSON
        actions = generate_random_sequence(random.randint(2, 6))
        
        # 反向描述
        nl_plan = reverse_describe(actions, client)
        
        dataset.append({
            "input": f"[Instruction]: 将以下计划转换为 JSON 动作序列。\n\n[Plan]:\n{nl_plan}\n\n[Output]:\n",
            "output": json.dumps(actions, ensure_ascii=False)
        })
        
        if i % 100 == 0:
            print(f"Generated {i}/{num_samples}")
    
    # 保存
    with open("data/translator_train.json", "w") as f:
        json.dump(dataset, f, ensure_ascii=False, indent=2)


if __name__ == "__main__":
    generate_dataset(5000)
