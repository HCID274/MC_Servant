# MC_Servant 项目掉转船头方向与运营策略

> **核心目标变更**: 从 "商业化盈利 (GMV)" 转向 **"求职作品集 (Portfolio) / MVP 验证"**
> **最后更新**: 2026-01-05

---

## 1. 战略调整：为简历服务的 PM 思维

项目的核心 KPI 不再是流水，而是 **DAU (日活)**、**Retention (留存)**、**System Stability (稳定性)** 和 **User Feedback (用户反馈)**。
服务器定位从 "商品" 转变为 **"测试实验室"**。

### 1.1 资源配置策略
- **Qwen-Flash (主力)**: 用于指令执行，保证逻辑准确性。
- **OpenRouter Free (氛围)**: 用于纯闲聊，断连或延迟可容忍，作为低成本兜底。

### 1.2 用户筛选漏斗
- **Sponsor (付费用户)**: 定义为 "高优先级算力赞助者"。
    - 权益：API 队列插队、更长记忆窗口。
    - 价值：验证商业闭环 MVP，体现用户付费意愿。
- **Tester (肝帝/测试者)**: 定义为 "RLHF 数据提供者"。
    - 任务：对 Bot 行为点赞/踩，完成特定测试任务 (如 "识别三种木材")。
    - 价值：收集 User Interaction Data 用于优化 Prompt。

### 1.3 简历亮点功能
- **实时状态面板 (Dashboard)**: 游戏内显示 Token 消耗、任务成功率、活跃 Agent 数 (体现可观测性)。
- **调试模式 (Debug Mode)**: 允许资深玩家看到 `[Thought]` 和 `[Action]` (体现 CoT 设计能力)。

## 2. 技术方向调整：最终定稿架构

### 2.1 问题背景
原 `PrerequisiteResolver` 使用 411 行硬编码 if-else 处理失败恢复，存在扩展性差、边缘情况难以覆盖的问题。

### 2.2 解决方案：Neuro-Symbolic 恢复策略
参考 **VillagerAgent (ACL 2024)**，采用动态策略选择机制：
- **Decompose**: 分解复杂任务。
- **Insert**: 插入前置任务。
- **Replan**: 换思路重规划。
- **Retry**: 瞬态错误重试。
- **Escalate**: 询问用户。

### 2.3 关键设计变更

#### A. 保护机制 (Cost Control)
- **简单错误优先 Retry**: 如 TIMEOUT 优先重试，不调用 LLM。
- **冷却时间**: 连续 LLM 调用设置 500ms 冷却。
- **回滚策略**: LLM 连续 3 次无效策略，自动降级回硬编码兜底。

#### B. 优先级调整 (Fast vs Slow Path)
**新策略**: **Symbolic Check (Fast Path) -> LLM DynamicResolver (Slow Path)**

- **理由**:
    1.  **速度与成本**: `INSUFFICIENT_MATERIALS` (缺木头) 等错误，符号规则毫秒级判定且 100% 准确，无需浪费 Token 问 LLM。
    2.  **确定性**: 合成表等死逻辑，Minecraft-Data 比 LLM 更可靠。
    3.  **泛化推导**: `PrerequisiteResolver` 支持基于命名规则的泛化推导（如 `*_planks` -> 4/次，`*_slab` -> 6/次），无需为每种木材手写规则，大幅降低维护成本。

#### C. Prompt 增强：注入 "经验"
在 DynamicResolver Prompt 中，必须注入 **Tag Knowledge (知识库)** 和 **Available Actions (动作集)**，而非仅注入状态。

- **痛点**: "樱花木板" vs "橡木木板" 的混淆。
- **解决**: 在 Prompt 中加入知识库提示:
    ```
    - 缺失物品: {missing_item}
    - 知识库提示: {missing_item} 可以用背包中的 {equivalent_items} 替代
    ```
    (利用 TagResolver 预处理后喂给 LLM)。

## 3. 运营与风控

- **Token 桶算法**: 限制每分钟交互次数，付费/免费用户分级。
- **预处理层**: 正则拦截无效指令，节省 Token。
- **日志留存**: 完整保存交互日志，作为优化 Prompt 和面试展示的宝藏。
